## What Is This?

**Voyage Board** — a local-first vacation coordination app. Groups create/join trips via shared codes and collaborate through chat, a pinned bulletin board, polls, and a shared itinerary. All data is client-side (TinyBase + localStorage). No backend.

## Stack

Next.js 15 (App Router) / React 19 (pinned 19.2.4) / TypeScript 5 (strict) / TinyBase v8 / Tailwind CSS v4 / Lucide React icons / CVA + clsx + tailwind-merge

## Architecture at a Glance

```
Pages (src/app/)  →  useTableRows/useRow hooks  →  TinyBase Store  ↔  localStorage
                     (useSyncExternalStore)          (store.ts)
```

- **StoreProvider** (`store-context.tsx`): Wraps app, hydrates from localStorage, persists on change. Manages user identity registry (`knownUsers`), `renameCurrentUser`.
- **Store helpers** (`store.ts`): Pure functions like `addMessage()`, `addPoll()`, `votePoll()` that mutate the store
- **7 tables:** trips, members, messages, polls, pollVotes, events, readMarkers
- **UI primitives:** Button, Input, Textarea in `/src/components/ui/` with CVA variants
- **Theme:** Dark-only, copper (#c2875a) accents, taupe (#1a1614) backgrounds, semantic CSS tokens

## Key Patterns

- **Semantic tokens only** — `bg-surface` not `bg-white`. All colors via CSS custom properties.
- **Store helpers** — Mutations go through pure functions in `store.ts`, not inline in components.
- **Reactive hooks** — `useTableRows(table, filterField, filterValue)` for lists, `useRow(table, id)` for singles.
- **Inline create forms** — Each feature page has form at top + list below. No separate create routes.
- **All client components** — `"use client"` on every component (TinyBase requires it).

## Known Constraints

- No backend, no auth — browser-based identity (UUID + name in localStorage)
- Multi-device sync available via WebSocket relay (TinyBase WsSynchronizer)
- Roles tracked, partially enforced (admin actions in member list, sole-admin leave protection)
- Poll options stored as JSON string (TinyBase limitation)
